## DemoGo 首页说服链重构

### 背景
用户要求首页按照连续说服链模型重新设计，而不是功能罗列。

### 四层结构

**第1层 — 核心价值承诺 (Hero)**
- 纯价值主张：一句"做完作品，直接扔个链接"完成产品定义
- 去掉了 badge、mockup 等干扰元素，让用户第一时间看到核心价值
- CTA 引导到"看看适不适合我"（第二层入口）

**第2层 — 痛点锚定 (Pain)**
- 三个真实场景："用 Cursor 做了原型不知道怎么给人看""设计师交付了 HTML 打不开""Windsurf 写了小工具想分享到群里"
- 深灰背景 + 卡片式呈现，让用户产生"这不就是我吗"的共鸣

**第3层 — 卖点强化 (Sell) ×3**
- 一屏一个核心卖点，左右交替布局（图+文）
- 卖点1: "不用配域名、不用搞 SSL、不用写 Dockerfile" → 标签：零部署/零配置/零命令行
- 卖点2: "链接丢出去，对方点开即用" → 标签：无需注册/跨平台/即时反馈
- 卖点3: "不管你用什么做，DemoGo 都能接住" → 标签：Cursor/Windsurf/VS Code/Replit/Claude Code

**第4层 — 决策收口**
- 适用人群（PM/设计师/运营 三卡片）
- 使用场景（客户演示/团队评审/社群测试/面试展示）
- 对比表格（自己部署 vs DemoGo：耗时/门槛/分享/费用四维度）
- 数据指标（500+/2,300+/97%）
- 最终 CTA + 消除顾虑："无需信用卡，随时取消。用完即走，没有任何绑定。"
