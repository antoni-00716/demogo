# DemoGo v0.1.26 开发方案：真实试用可信增强版

日期：2026-05-18

## 版本目标

v0.1.26 的目标是让 DemoGo 更适合开放给真实用户试用：正常项目能顺利发布，高风险内容能被拦截，疑似风险能进入后台复核，用户能看懂失败原因和下一步怎么改。

本版不是新增大范围托管能力，而是补齐“公开链接平台”的可信度和运营处理能力。

## 本版范围

### 任务一：内容安全生产化增强

要做：

- 保留 v0.1.25 的本地规则初筛。
- 增加内容安全决策层，把检查结果统一归类为：
  - `passed`：允许发布。
  - `blocked`：明确拦截。
  - `review_required`：进入人工复核，不直接发布。
  - `failed`：检查异常，默认不放行。
- 增加第三方内容安全接入配置和适配接口：
  - 暂不硬依赖阿里云/腾讯云 SDK。
  - 先做标准接口层，允许通过环境变量启用外部服务。
  - 没有配置第三方服务时，仍使用本地规则。
- 为后续文本审核、图片审核预留字段和结果结构。

不做：

- 不承诺当前已完成正式第三方智能审核。
- 不做视频、音频内容审核。
- 不做完整人工内容审核平台。

### 任务二：管理后台人工复核闭环

要做：

- 内容检查记录增加运营处理状态：
  - `pending`：待处理。
  - `confirmed_violation`：确认违规。
  - `false_positive`：误判。
  - `resolved`：已处理。
- 增加管理员备注字段。
- 增加处理人和处理时间。
- 新增后台接口：处理内容检查记录。
- 后台内容检查列表展示处理状态。
- 后台详情抽屉支持修改处理状态和备注。
- 后台首页突出显示待处理内容检查数量。

不做：

- 不做“管理员放行后自动发布”的完整工作流。
- 不做复杂权限分级。

### 任务三：用户端失败体验优化

要做：

- 对失败原因做分类展示：
  - 内容风险。
  - 疑似风险待确认。
  - 项目结构问题。
  - 后端能力暂不支持。
  - 套餐额度不足。
  - 构建失败。
  - 服务器处理异常。
- 内容风险场景展示更清楚的下一步：
  - 为什么不能生成链接。
  - 需要删除或改写哪些内容。
  - 可复制给 AI 编程工具的修改指令。
- 疑似风险场景说明“需要平台确认后才能发布”。
- AI 发布接口返回更清楚的失败信息，方便 Codex/Cursor/Claude Code 等工具直接转述给用户。

不做：

- 不做三端大规模视觉重构。
- 不做新的 CLI/MCP/Codex Skill 能力。

## 数据结构调整

`content_reviews` 增加字段：

- `resolution_status`
- `admin_note`
- `handled_by`
- `handled_at`

JSON 记录同样增加：

- `resolutionStatus`
- `adminNote`
- `handledBy`
- `handledAt`

默认规则：

- `blocked`、`review_required`、`failed` 默认 `resolutionStatus=pending`。
- `passed` 默认 `resolutionStatus=resolved`。

## 后端接口调整

新增：

- `POST /api/admin/content-reviews/:id/status`

请求：

```json
{
  "resolutionStatus": "confirmed_violation",
  "adminNote": "已确认为高风险导流页面"
}
```

返回：

```json
{
  "review": {}
}
```

保留：

- `GET /api/admin/content-reviews`
- `GET /api/admin/overview`

增强：

- 概览增加 `pendingContentReviewResolutions`。
- 内容检查列表支持按 `resolutionStatus` 过滤。

## 用户端文案原则

不能说：

- “策略”
- “合规红线”
- “连带责任”
- “内部风控”
- “实验能力”

应该说：

- “这个页面暂时不能公开分享”
- “页面里包含可能误导用户的内容”
- “请先修改后再重新上传”
- “如果只是演示，请删除真实付款、私聊导流或敏感信息收集”

## 验收标准

- 正常项目可以发布。
- 明显违规项目不能发布。
- 疑似风险项目不能发布，并进入后台待处理。
- 后台能查看内容检查记录。
- 后台能修改处理状态和备注。
- 用户端能看到清楚的失败原因和 AI 修改指令。
- AI 发布接口同样返回清楚的内容风险结果。
- `server npm run check` 通过。
- `server npm run test:smoke` 通过。
- `web npm run lint` 通过。
- `web npm run build` 通过。
- 生成 v0.1.26 服务端包和运维脚本包。

## 已知边界

- 本版仍不支持后端项目长期运行。
- 本版仍不支持数据库自动分配。
- 本版仍不支持真实支付、订单、登录系统托管。
- 第三方内容安全如果没有配置，不影响本地规则初筛运行。
