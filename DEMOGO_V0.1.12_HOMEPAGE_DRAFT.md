## 1. 首页目标

首页不是功能说明页，而是试运营转化页。

目标用户进入首页后，需要在 30 秒内明白：

- DemoGo 解决什么问题；
- 为什么不用截图、录屏、zip 或国外部署平台；
- DemoGo 适合什么场景；
- DemoGo 现在能做什么；
- 如何开始发布第一个 Demo。

核心判断：

> DemoGo 不是 Vercel / Netlify 的低配替代品，而是面向国内 AI 生成项目试用验证场景的低门槛发布工具。

## 7. 为什么选择 DemoGo

标题：

```text
别人给你部署工具，DemoGo 给你试用链接。
```

说明：

```text
Vercel、Netlify 这类平台很强，适合开发者做正式工程部署。
但很多国内试用场景的真正问题不是“能不能部署”，
而是客户、同事、领导、投资人能不能稳定打开。

DemoGo 面向的是 AI 生成项目的快速试用验证：
上传项目包，生成国内用户更容易打开的试用链接。
```

对比表：

| 方式 | 更适合谁 | 主要麻烦 |
|---|---|---|
| Vercel / Netlify | 开发者、正式工程部署 | Git、构建、环境变量、域名配置；国内试用访问可能不稳定 |
| 自己买服务器部署 | 技术团队 | 服务器、Nginx、SSL、域名解析、备案、运维 |
| 截图 / 录屏 / zip | 临时沟通 | 不能真实点击试用，反馈价值低 |
| DemoGo | AI 产品验证、客户演示、活动页试运营 | 上传 zip，自动检测，生成试用链接和转发文案 |

强化句：

```text
DemoGo 把部署、服务器、域名、检测和转发这些麻烦，压缩成一次上传。
```

## 8. 三步生成试用链接

标题：

```text
从 AI 生成项目到可试用链接，只需要三步。
```

步骤：

### 1. 上传项目包

```text
把 AI 工具生成的网页或前端项目打成 zip 上传。
```

### 2. DemoGo 自动体检

```text
检查入口页面、构建配置、表单字段、本地 API 和发布风险。
```

### 3. 生成链接并转发

```text
得到可访问 Demo 链接，一键复制转发文案，发给客户、同事或试用用户。
```

## 9. 发布前基础体检

标题：

```text
不只是生成链接，还会在发布前做基础体检，提示可能影响试用的风险。
```

说明：

```text
当前 DemoGo 是规则体检基础版，不是大模型 AI 判断，也不是完整业务功能测试。
它会尽量提前发现“页面能打开，但某些交互可能不能用”的问题。
```

能力：

| 能力 | 用户价值 |
|---|---|
| 检测入口页面 | 判断页面能否打开 |
| 检测表单字段 | 识别报名、预约、反馈类页面 |
| 检测本地 API | 提醒 `/api/register`、`/api/submit` 等接口发布后可能不可用 |
| 检测构建配置 | 判断是否能自动生成前端页面 |
| 提示风险 | 减少发给别人后才发现不能用的尴尬 |
| 生成 AI 修复提示 | 方便回到 AI 编程工具继续修改 |
| 生成转发文案 | 帮用户更自然地把链接发给试用对象 |

## 12. 支持边界与合规提醒

标题：

```text
当前最适合前端页面和静态 Demo。
```

说明：

```text
如果项目依赖自定义后端、数据库或真实支付，DemoGo 会在检测时提示风险。
```

支持：

```text
静态 HTML
dist / build 前端项目
常见 React / Vue / Vite 源码项目
活动页、展示页、报名页前端
```

暂不支持：

```text
完整后端服务
数据库
Docker
真实支付
自动接管任意自定义 API
```

合规提醒：

```text
DemoGo 适合测试、演示和试用验证。
如果后续要绑定独立域名、做正式商用网站或长期运营，
仍需根据实际业务办理域名、备案和相关合规事项。
```
