# Material: DemoGo - AI编程产物一键发布与试用平台

## 1. Overview
- DemoGo 是一个面向 AI 编程时代的项目一键发布与试用平台，v0.5.0，域名 https://demogo.cn
- 核心价值："AI 生成代码 → 一键发布 → 马上试用"的开发验证闭环
- 已支持静态网页、前端源码项目（React/Vue/Vite）、Node.js 单服务应用、MySQL 试用数据库
- 配套 CLI/MCP/Skill/Agent API 四层 AI 工具集成体系

## 2. Background
- 2024-2026 年 AI 编程工具爆发式增长（Codex、Cursor、Claude Code、Trae 等）
- 大量非技术用户（产品经理、创业者、运营人员）开始用 AI 生成软件产品
- "AI 做出来的东西如何让他人试用"成为新瓶颈——不会部署、无法分享
- 国内市场缺乏面向 AI 编程产物的专用发布平台

## 3. Key Info
- 当前版本：v0.5.0
- 线上域名：https://demogo.cn（ICP备案：鄂ICP备2026023999号）
- CLI 工具：@demogo-cn/cli@0.5.0 已发布 npm
- 17 种项目类型自动识别（React/Vue/Vite/Next.js/Nuxt/Astro/Express/NestJS 等）
- Docker 容器运行 Node.js 单服务应用，MySQL 试用数据库自动分配
- 内容安全审查（12类风险规则），自动表单收集
- Free/Lite/Pro 三级套餐，验证商业模型

## 4. Evidence
- 案例：用户通过 AI 工具生成产品落地页 → DemoGo 一键发布 → 获得试用链接 → 分享给客户验证
- 案例：创业者用 AI 做报名页 → DemoGo 自动接管表单收集 → 收到真实报名数据
- 案例：开发者用 AI 做全栈应用 → DemoGo 识别 Node.js+MySQL → Docker 运行 + 分配数据库 → 完整试用

## 5. Analysis
- 市场定位：AI 编程产物发布的"最后一公里"，差异化明显
- 竞品差异：Vercel/Netlify 面向开发者，DemoGo 面向非技术用户和 AI 工具使用者
- 技术壁垒：多层级项目识别引擎 + Docker 隔离运行时 + 四层 AI 工具集成

## 6. Outlook
- 近期：v0.5.1~v0.6.0 扩展运行态能力至完整应用部署
- 中期：成为 AI 编程生态的"标准发布协议层"
- 远期：企业级私有部署、行业解决方案
