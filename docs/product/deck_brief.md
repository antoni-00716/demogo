# DemoGo OPC 比赛 PPT — 整体简报

## 受众画像

| 维度 | 描述 |
|------|------|
| 身份 | OPC 大赛评委（技术专家、投资人、产业代表） |
| 认知基线 | 了解 AI 编程工具趋势（Cursor/Copilot/Bolt），但对部署环节认知模糊 |
| 关注点 | 技术创新性 > 市场空间 > 商业可行性 > 团队执行力 |
| 潜在抗拒 | "AI 编程工具已经很多了，这个有什么不同？""技术门槛不高吧？""能赚钱吗？" |
| 决策权重 | 创新性 35% / 实用性 25% / 市场前景 20% / 团队 20% |

## 场景

| 维度 | 描述 |
|------|------|
| 场景 | OPC 大赛项目路演 |
| 时长 | 约 8-10 分钟演讲 + 答辩 |
| 屏幕 | 投屏展示（16:9），阅读距离 3-8 米 |
| 信息密度 | 每页 1 个核心观点 + 视觉支撑，避免文字堆砌 |
| 基调 | 专业自信、数据驱动、技术可信、适度野心 |

## 核心目标

> 让评委在 10 分钟内完成三个认知跃迁：
> 1. "这是 AI 编程生态中真实存在的痛点" → 认同问题
> 2. "DemoGo 的技术方案有壁垒" → 认可方案
> 3. "这个赛道值得投，团队能做成" → 认可价值

## 核心张力

```
AI 编程工具让"写代码"民主化
          ↓
但"让代码变成可用的产品"仍然是瓶颈
          ↓
DemoGo = AI 编程产物的最后一公里
```

**一句话张力:** 人人能写 AI 代码，但不是人人能让代码跑起来被世界看见。

## 叙事弧线

```
Hook (P3-4)     →  "你写的 AI 代码，部署了吗？"
Solution (P5-7) →  "DemoGo 怎么做"
Proof (P8)      →  "已经跑起来了"
Market (P9-10)  →  "为什么现在是最好的时机"
Business (P11-14) → "怎么持续做下去"
Team + Close (P15-17) → "为什么是我们"
```

## 设计原则

| 原则 | 说明 |
|------|------|
| 深蓝 + 青色主色调 | Tech 信任感 + AI 活力感 |
| 每页一个核心数字/图表 | 可视化优先，降低认知负荷 |
| 标题即观点 | 不看正文也能理解全页论点 |
| 留白大方 | 路演投屏需要呼吸感 |
| 动效服务于叙事 | 不做无意义动效，只在需要强调对比/流程时使用 |

## 成功标准

| 标准 | 衡量方式 |
|------|----------|
| 3 秒可读 | 任意一页投屏后，评委 3 秒内抓住核心信息 |
| 问题认同 | P3-P4 讲完后评委产生"确实如此"的心理反应 |
| 壁垒感知 | P5-P7 讲完后评委认为"这不只是简单包装" |
| 数据信服 | 市场/财务数据有来源、有逻辑、经得起追问 |
| 记忆锚点 | 评委离场后能记住 "AI代码→一键发布→马上试用" + 1-2 个技术亮点 |
