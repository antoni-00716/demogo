# DemoGo v0.1.11 任务记录

## 版本定位

v0.1.11 是“用户试用闭环版”，重点不是继续增加大功能，而是把用户从发布、理解额度、申请升级、复制转发文案到运营后台处理申请的流程打通。

## 已实现

- 用户端删除无效“订单记录”，改为“本月发布记录”。
- 用户端套餐区改为“套餐与额度”，使用 Free / Lite / Pro 卡片说明权益。
- 用户端展示最近升级申请状态。
- 用户端新增发布记录 API 和列表，说明为什么发布额度被扣减。
- 复制转发文案改为真正一键复制；只有复制失败时才显示手动复制文本框。
- 管理后台“用户列表”改为只读，不再直接编辑套餐。
- 管理后台套餐开通统一通过“升级申请”处理。
- 拒绝升级申请仍使用页面内输入，不使用浏览器 prompt。
- 后端新增 `GET /api/deploy-events`，复用已有 Demo 发布事件，不新增表。
- 后端服务版本升级到 `0.1.11`。

## 本版不做

- 不接支付。
- 不做正式订单系统。
- 不做套餐有效期和自动降级。
- 不接大模型。
- 不做表单托管。
- 不做 Docker / 后端项目托管。

## 自测重点

- 注册、登录、发布、更新、下线、恢复、删除。
- 项目检测规则报告。
- 反馈提交和后台处理。
- 升级申请提交、后台开通、后台拒绝。
- 发布记录 API 能正确展示发布和更新记录。
- 用户端复制转发文案不跳转、不默认展示文本框。
- 管理后台没有浏览器 `prompt`。
- 构建包包含 v0.1.11 后端、前端和运维脚本。

## 后续建议

- v0.1.12 优先做真实用户试用数据面板、失败原因归类、操作日志可视化。
- 中期逐步把大文件 `server.js` 拆成路由、服务和数据访问层。
- 表单托管、AI 体检增强、支付订单放到后续独立版本，不挤进本版。
