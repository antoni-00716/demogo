# DemoGo v0.2.3 开发方案：真实试用与异步发布合并版

日期：2026-05-19

## 版本定位

v0.2.3 合并原计划的 v0.2.3 和 v0.2.4，不再单独开发 v0.2.4。

本版目标不是扩大托管范围，而是把真实试用和发布体验做稳：

- 用户上传后不要长时间卡在一个请求里。
- 发布过程要能看到进度和失败步骤。
- AI 工具、CLI、MCP 继续复用同一条发布链路。
- 继续明确支持与不支持边界，不把后端托管、数据库托管包装成当前能力。

## 本版任务

### 1. 异步发布任务

要做：

- 新增发布任务接口：
  - `POST /api/deployment-jobs`
  - `GET /api/deployment-jobs/:id`
  - `POST /api/demos/:id/deployment-jobs`
- 网页端创建和更新项目改为先创建任务，再轮询任务状态。
- 发布任务返回：
  - 排队中
  - 生成中
  - 成功
  - 失败
  - 检查报告
  - 内容检查结果
  - 发布步骤
  - 最终链接
- 后端任务继续复用原发布逻辑，不绕过项目检测、额度检查、内容安全检查、表单识别和发布记录。

不做：

- 不引入 Redis/BullMQ。
- 不做独立 Worker 服务。
- 不做 Docker 构建隔离。

### 2. 用户端发布体验优化

要做：

- 上传后显示处理进度。
- 失败时展示具体失败步骤和可理解原因。
- 任务超时时提示用户稍后在项目列表或发布记录查看。
- 更新项目也走同样的任务进度，不再让用户感觉页面卡住。

### 3. 真实试用稳定性补强

要做：

- 保留同步发布 API，确保 CLI、MCP、Agent API 兼容。
- CLI/MCP/Skill 版本升级到 `0.2.3`。
- 部署脚本、回滚脚本、上传脚本、打包脚本升级到 `0.2.3`。
- 真实项目样本继续覆盖：
  - 单 HTML
  - `dist`
  - `build`
  - 表单页
  - 计算器
  - React/Vite 源码
  - Vue/Vite 源码
  - 缺 build 命令源码
  - 后端项目
  - SSR 项目
  - 包含敏感文件
  - 内容风险项目

## 当前明确不支持

- Node.js/Python 后端长期运行。
- Express、FastAPI、Flask、Django、NestJS 等后端服务托管。
- Next/Nuxt/Remix SSR runtime。
- 数据库自动分配。
- 支付、订单、登录系统后端。
- WebSocket、AI Proxy。
- `.rar` 项目包。

## 技术方案

### 后端

发布核心拆成两个可复用函数：

- `performCreateDeployment`
- `performUpdateDeployment`

同步接口继续直接调用这两个函数。

异步任务接口先创建任务记录，然后后台执行同一套发布函数。任务状态保存在 `deployment-jobs.json`。在线上 MySQL 兼容层里，短期复用 `audit_logs` 表保存任务 JSON，后续任务量上来后应独立建 `deployment_jobs` 表。

### 前端

用户端发布按钮不再直接等待最终发布结果，而是：

```text
选择项目包
  -> 创建发布任务
  -> 轮询任务状态
  -> 更新步骤进度
  -> 成功展示链接
  -> 失败展示原因和检查报告
```

### AI 发布链路

CLI、MCP、Agent API 保持原同步接口，避免破坏当前 AI 工具发布能力。后续如果 AI 发布也需要任务式体验，再扩展 Agent 任务接口。

## 验收标准

- 后端语法检查通过。
- 后端烟雾测试通过。
- 前端 lint 通过。
- 前端生产构建通过。
- CLI/MCP 基础检查通过。
- 真实项目样本生成通过。
- 用户端创建项目和更新项目可以看到任务进度。
- 发布失败不再只显示笼统失败，而是能看到检查报告或失败步骤。
- v0.2.3 六个交付包生成完整。
