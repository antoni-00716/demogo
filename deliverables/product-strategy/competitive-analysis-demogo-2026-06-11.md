## 📌 TL;DR（执行摘要）

- **核心发现**：DemoGo 填补了「AI编程工具生成项目→快速分享试用」这个空白地带。目前没有任何一个竞品专门定位这个场景。v0/Bolt/Lovable 是"从0到1的生成器+部署"，而 DemoGo 是"已有成品后的分享发布层"。
- **最大机会**：Cursor 100万+付费用户 + Claude Code 爆发式增长（6个月达$10亿年化），这些用户每天都在生成项目但缺少一键部署到可访问链接的解决方案。中国开发者AI工具渗透率仅30% vs 美国91%，增长空间巨大。
- **最被忽视的竞品**：Tiiny.host（理念高度相似——上传→秒得分享链接）、EdgeOne Pages Drop（国内完全免费静态托管）、Zeabur（国内可访问的PaaS，已推AI Agent）
- **已有关键能力**：DemoGo **已有 CLI + MCP 协议**，AI 编程工具（Cursor/Claude Code/Windsurf）可通过自然语言直接调用发布——这是比 Tiiny.host/Pages Drop/Zeabur 都更原生 AI 的差异化能力
- **建议行动**：重点推广「DemoGo CLI + MCP = AI工具原生发布层」的市场心智，而非"从零建插件"；搭配 Cursor/Claude Code 插件生态深化（MCP 是桥梁，插件是体验升级）；国内市场先行占领"AI项目国内发布"心智；定价建议 $15-20/月主力点。

### 2.2 DemoGo 核心差异定位（SWOT）

**优势（可防守的壁垒）**
- 唯一支持「上传已有项目→生成试用链接」的定位
- 3-7天有效期的轻量试用，无需域名、备案、服务器
- 面向 AI 编程工具用户（Cursor/Claude Code/Windsurf），这不是传统部署平台关注的人群
- 国内可访问，无需翻墙
- **用户研究核心发现**：国内访问所有海外平台都是2/10体验，Vercel账单暴涨5-10倍是公认痛点

**劣势**
- 不支持数据库和后端服务 → 限制了能展示的项目类型
- 只做"前发布"阶段（试用），不做"生产级"托管 → 用户后期仍需迁移
- 功能非常轻量，容易被上下游吞并

**机会**
- AI编程工具爆发 → Cursor/Claude Code/Windsurf用户大量产生需要分享的项目（Cursor 100万+付费用户、Claude Code $10亿年化）
- 当前竞品都聚焦"AI生成"，而非"AI生成后的发布"
- **AI工具原生集成是最大壁垒**：v0/Bolt/Lovable自己就是AI工具，不具备这个集成点；Tiiny.host和Pages Drop也不具备
- 中国市场空白——Vercel/Netlify 在国内访问困难，中国AI编程渗透率仅30% vs 美国91%
- Vibe Coding部署子市场年增长率 **36.79%**，2040年预计 $3,250亿

**威胁**
- v0 已绑定 Vercel 部署（最成熟路径），用户习惯已形成
- Bolt.new 内置全栈+部署一站式体验非常顺滑
- AI 编程工具自身可能加发布功能（Cursor 已有 Preview 功能）
- 腾讯 EdgeOne Pages 在国内免费且功能强大（15万+用户）
- Tiiny.host 产品理念高度相似，若开始支持动态应用将是劲敌

### 4.2 三大 Tier 1 用户痛点

**痛点1：国内访问体验极差**
- v0/Bolt/Lovable/Vercel/Netlify 国内评分均仅 **2/10**
- Lovable 必须翻墙
- 中国市场开发者 AI 工具渗透率仅 30%（vs 美国 91%），部分原因就是工具不可用

**痛点2：定价不可预测**
- Vercel 用户投诉频率最高：账单突然飙升 **5-10 倍**
- Reddit/HN 关于"Vercel bill shock"的帖子持续高热度
- 开发者对部署服务的支付意愿上限在 **$20/月心理关口**

**痛点3："最后一公里"部署断裂**
- arXiv 学术论文证实：AI 编程工具用户在"代码→上线"环节严重断裂
- Cursor、Claude Code 等工具不提供原生部署方案
- 用户需要手动完成：导出代码→配置CI/CD→域名→DNS→备案→上线

### 4.3 各用户群的差异化机会

| 用户群 | 当前痛点 | DemoGo 机会 |

## 6. 差异化机会

### 6.1 核心壁垒方向：AI 编程工具原生的最后一公里

1. **AI 工具原生集成（P0 — 最高优先级）**
   - ✅ **已有 CLI + MCP 协议**，Cursor/Claude Code/Windsurf 等 AI 编程工具可直接通过自然语言调用 `deploy` 命令
   - 这是 v0/Bolt/Lovable/Tiiny.host/Pages Drop 都不具备的能力——它们没有 CLI/MCP 级 AI 集成
   - **下一步**：基于 MCP 协议包装为 Cursor/Windsurf 插件，降低用户学习成本，提升"一键"体验
   - 推广「自然语言即可发布」的市场心智：用户在 Cursor 里说"帮我发布到 DemoGo"就能直接执行
   - Cursor 100万+付费用户 + Claude Code 爆发式增长的庞大付费用户基础

2. **项目类型识别与智能构建**
   - 自动识别上传项目的框架（React/Vue/Next.js/Flask 等）
   - 自动安装依赖、构建、启动
   - 这是"上传已有项目"场景的核心体验

3. **试用体验增强**
   - 3-7天有效期本身是差异点，可增加"试用链接的交互层"
   - 例如：在试用页面加水印、记录访客行为、收集反馈
   - 从"部署工具"进化为"试用分发平台"

### 6.2 追平领域

- **数据库支持**：即使是 Demo 阶段，有数据库的项目占比也很大。考虑集成 SQLite/临时 PostgreSQL/Turso
- **自定义域名**：用户可能希望在试用阶段绑定自己域名（如 demo.myproject.com）
