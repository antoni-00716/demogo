# DemoGo 官网上线清单

这份清单用于备案通过后，把 `demogo.cn` 第一版官网上线到阿里云 ECS。当前阶段先上线官网、内测入口和合规页面，不开放大规模用户发布。

## 备案通过前

- 确认 `demogo.cn` 域名实名认证主体和备案主体一致。
- 完成 ICP 备案资料提交、短信核验和管局审核。
- 不要把 `demogo.cn` 正式解析到中国内地 ECS 对外访问。
- 在 ECS 上预先安装运行环境：Nginx 或 Caddy、Docker、证书工具、基础防火墙。
- 准备站点目录，例如 `/var/www/demogo.cn`。
- 准备要上传的官网文件：
  - `index.html`
  - `app.html`
  - `admin.html`
  - `terms.html`
  - `privacy.html`
  - `content-policy.html`
  - `assets/demogo-hero.png`

## 备案通过当天

- 在阿里云 DNS 添加解析：
  - `@` A 记录指向 ECS 公网 IP。
  - `www` CNAME 到 `demogo.cn`，或 A 记录指向 ECS 公网 IP。
- 把官网文件放到站点目录。
- 配置 Web 服务并确认 80 端口可访问。
- 申请并安装 HTTPS 证书。
- 配置 HTTP 自动跳转 HTTPS。
- 在首页底部填入真实 ICP 备案号，并链接到工信部备案系统。
- 检查移动端页面、邮件链接、协议链接和图片加载。
- 检查 `/app.html` 用户控制台原型。
- 检查 `/admin.html` 管理后台原型。正式上线时后台应加登录和 IP 白名单，不要作为公开入口展示。

## 上线后 30 天内

- 到全国互联网安全管理服务平台提交公安联网备案。
- 备案通过后，在网站底部添加公安备案号和图标链接。
- 建立举报与下线处理流程，至少保留一个有效邮箱。

## 内测开放前

- 明确免费版限制：1 个在线 Demo、3-7 天有效、每月 3 次发布/更新。
- 明确早期付费口径：轻量版 19.9 元/月，Pro 版 39 元/月。
- 给每个内测用户人工建立项目记录，先不要急着做完整支付。
- 先支持静态前端项目，暂不支持数据库、后端长驻服务和高风险内容。
- 每个 Demo 必须支持手动下线和到期自动下线。
- 保留访问日志、发布记录、用户联系方式和项目来源，便于风控与排障。

## 第一批内测用户筛选问题

1. 你现在使用什么 AI 编程工具？
2. 你要发布的 Demo 是什么类型？
3. 现在卡在部署的哪一步？
4. 这个链接要发给谁试用？
5. 你希望链接保留几天？
6. 如果服务稳定，你能接受 9.9 元/次或 19.9 元/月吗？
