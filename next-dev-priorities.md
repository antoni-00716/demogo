# DemoGo 下一阶段开发优先级

当前 DemoGo 已跑通上传发布闭环，但还不能公开开放。下一阶段优先级必须先控风险，再补商业能力。

## P0：公开前必须做

1. `/api/deploy` 加临时鉴权
   - 最小做法：上传接口要求一个发布密码。
   - 页面上传时带上密码。
   - 防止公网任何人上传文件。
   - 当前实现方式：`DEMOGO_DEPLOY_TOKEN`。

2. 上传接口限流
   - IP 级别限制。
   - 单次上传大小限制已做，还需要频率限制。

3. Demo 下线接口
   - 用户端可以下线自己的 Demo。
   - 管理员下线任意 Demo 待实现。

4. Demo 元数据存储
   - 先用 SQLite 或 JSON 文件。
   - 记录 slug、文件大小、状态、创建时间、到期时间、来源 IP。

5. 到期自动清理
   - 免费 Demo 默认 3-7 天。
   - 定时任务删除过期目录。

## P1：内测需要做

1. 简单登录
   - 先用手机号/邮箱还不急。
   - 早期可以用邀请码或访问密码。

2. 免费额度控制
   - 1 个在线 Demo。
   - 每月 3 次发布/更新。
   - v0.1.1 已实现。

3. 管理后台真实数据
   - Demo 列表。
   - 用户列表。
   - 运营指标。
   - v0.1.1 已接入概览数据，真实管理操作待实现。

4. 构建日志
   - 当前只支持已构建项目。
   - 后续支持源码包时必须展示日志。

## P2：商业化需要做

1. 套餐系统
   - Free / 轻量版 / Pro。
   - 在线 Demo 数、发布次数、有效期。

2. 订单系统
   - 早期人工开通。
   - 后续接微信/支付宝。

3. 访问统计
   - Demo 访问次数。
   - 最近访问时间。
   - 超限提醒。

4. AI 工具接入
   - CLI。
   - MCP Server。
   - Codex Skill。

## 下一步建议

v0.1.2 已完成到期自动清理、管理员下线、上传限流和审计日志。

下一步不要急着做支付。先做：

```text
构建 Worker + 支持 Vite/React/Vue 源码包
```

这会显著提升真实 AI 编程项目的上传成功率。
