# DemoGo v0.2.7 计划：CLI 防误打包与 AI 发布体验优化版

## 版本目标

v0.2.7 解决真实 AI 发布测试中暴露的误操作：AI 在桌面目录直接运行 CLI，导致把整个桌面打包并超限。

本版本不新增后端托管、数据库托管、支付、登录系统托管等长期能力。

## 本版范围

1. CLI 防误打包
   - 拦截用户主目录、Desktop、Downloads、Documents、OneDrive、桌面、下载、文档等高风险目录。
   - 提示用户新建干净文件夹，只放要发布的项目文件。
   - 支持继续使用 `--dir <项目目录>` 指定真正项目目录。

2. 超限提示优化
   - 文件数量超过上限或体积超过上限时，不只提示“超限”。
   - 明确提醒“你可能在错误目录执行了发布”。
   - 引导用户切换到项目目录，或把单 HTML 文件放入干净目录后发布。

3. MCP 同步
   - MCP 使用同样的目录保护逻辑。
   - 避免 AI 工具绕过 CLI 后继续在错误目录打包。

4. 用户端和 Skill 指令优化
   - 发布前先确认当前目录只包含本项目。
   - 不要在桌面、下载、文档、OneDrive 或用户根目录直接执行发布。
   - 单 HTML 文件不需要改名为 `index.html`，应放入干净目录后发布。

## 验收标准

- CLI 在桌面/下载/文档/用户根目录执行时会被拦截，并给出可理解提示。
- CLI 在干净目录中仍可正常打包发布。
- 单 HTML 文件仍然不需要手动改名。
- 后端检查、烟雾测试、前端 lint、前端生产构建通过。
- CLI/MCP 语法检查通过。
- npm dry-run 可生成 `@demogo-cn/cli@0.2.7`。
